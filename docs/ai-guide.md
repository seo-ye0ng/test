# AI Guide

## 목적

이 문서는 AI가 이 저장소를 분석할 때 참고해야 하는 기준을 정의합니다.

## 이 저장소의 역할

이 저장소는 디자인 시스템과 하네스 엔지니어링을 위한 자료를 관리합니다.  
AI는 이 저장소를 기반으로 디자인 토큰, 컴포넌트 명세, 에셋, 문서 규칙을 분석하고 개선 방향을 제안할 수 있습니다.

## AI가 먼저 확인해야 할 파일

1. `README.md`
2. `docs/token-rule.md`
3. `docs/naming-rule.md`
4. `tokens/primitives.json`
5. `tokens/semantic.json`
6. `components/*.md`

## AI에게 기대하는 작업

- 폴더 구조 분석
- 토큰 구조 분석
- Primitive Token과 Semantic Token 관계 분석
- 컴포넌트별 필요한 토큰 추정
- 누락된 컴포넌트 명세 제안
- 네이밍 규칙 위반 항목 탐지
- 개발자가 사용할 수 있는 CSS 변수 또는 코드 구조 제안

## AI 분석 시 주의사항

- 바로 코드를 수정하지 말고 먼저 분석 결과를 제시합니다.
- 추정한 내용은 추정이라고 명확히 표시합니다.
- 토큰 이름을 변경할 때는 기존 사용처에 미치는 영향을 함께 설명합니다.
- 컴포넌트 명세가 부족하면 필요한 질문을 먼저 정리합니다.
- 디자인 토큰과 컴포넌트 명세를 분리해서 분석합니다.

## 추천 프롬프트

```text
이 저장소는 하네스 엔지니어링을 위한 디자인 시스템 저장소입니다.
README.md, docs, tokens, components 폴더를 기준으로 전체 구조를 분석해주세요.

먼저 다음 순서로 답변해주세요.

1. 현재 폴더 구조 요약
2. 각 파일의 역할 추정
3. 디자인 토큰 구조 분석
4. 컴포넌트 명세의 충분성 평가
5. AI가 읽기 좋은 구조로 개선할 점
6. 개발 코드로 변환하기 전 필요한 추가 정보
```

## 금지 사항

- 민감정보, API Key, 인증 토큰을 저장소에 포함하지 않습니다.
- 원본 디자인/코드를 무단 복제하기 위한 분석에는 사용하지 않습니다.
- 불확실한 내용을 확정된 사실처럼 작성하지 않습니다.
